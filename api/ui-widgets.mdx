## Overview

The `ui.widgets` module provides factory functions for creating Tkinter widgets with consistent styling across the application. All widgets follow a unified dark theme with customizable colors, fonts, and borders.

**Source:** `~/workspace/source/ui/widgets.py`

## Theme Constants

Widgets use these constants from `constants.py`:
- `BG` - Main background (darkest)
- `BG2` - Panel background
- `BG3` - Input field background
- `ACCENT` - Primary accent color (blue)
- `ACCENT2` - Action color (orange/red)
- `SUCCESS` - Success color (green)
- `TEXT` - Primary text color
- `TEXT2` - Secondary text color
- `BORDER` - Border color

---

## Global Styling

### apply_style

```python
def apply_style(root)
```

Configures the global ttk theme for the application. **Must be called once** during main window initialization.

**Customizes:**
- Notebook (tab container)
- Tabs
- Frame & LabelFrame
- Combobox
- Checkbutton
- Scrollbar

<ParamField path="root" type="tk.Tk">
  The root Tkinter window
</ParamField>

**Example:**
```python
root = tk.Tk()
apply_style(root)
```

### make_scrolled_frame

```python
def make_scrolled_frame(parent) -> tk.Frame
```

Creates a frame with vertical scrolling support using Canvas + Scrollbar.

<ParamField path="parent" type="tk.Widget">
  Parent container
</ParamField>

**Implementation:**
- Uses `Canvas` with embedded `Frame`
- Vertical `ttk.Scrollbar`
- Auto-recalculates scroll region on `<Configure>` event
- Mouse wheel scrolling enabled via `<MouseWheel>` binding

**Returns:** `tk.Frame` - The **inner frame** where widgets should be placed

**Usage:**
```python
scrolled = make_scrolled_frame(parent)
make_label(scrolled, "Content 1").pack()
make_label(scrolled, "Content 2").pack()
# ... more content
```

<Info>
  Useful for tabs with long vertical content (VLANs, QoS, Routing).
</Info>

## Theming Philosophy

<AccordionGroup>
  <Accordion title="Why factory functions?">
    - **Centralized styling**: Change all button colors by editing `make_button()` once
    - **Consistency**: Every widget across all tabs shares the same visual language
    - **Rapid prototyping**: Build complex UIs with minimal boilerplate
  </Accordion>

  <Accordion title="Color scheme">
    The app uses a **dark theme** inspired by GitHub Dark:
    - Background layers: BG → BG2 → BG3 (darkest to lighter)
    - Primary accent: Blue (`#58a6ff`) for interactive elements
    - Action color: Orange/red for destructive or primary actions
    - Success: Green for confirmations
    - Text hierarchy: TEXT (bright) vs TEXT2 (dimmed)
  </Accordion>

  <Accordion title="Mouse wheel scrolling">
    The `make_scrolled_frame()` binds `<MouseWheel>` globally with `bind_all()`. This may conflict with other scrollable widgets. For production use, consider binding only to the canvas or using a more scoped event handler.
  </Accordion>
</AccordionGroup>
