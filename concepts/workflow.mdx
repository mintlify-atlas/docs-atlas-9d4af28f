## Overview

SECURE-BRANCH-lac uses a **9-tab workflow** that guides you through all aspects of switch configuration. Each tab builds on the previous ones, creating a complete configuration by the time you reach the Execution tab.

<Info>
You don't need to complete all tabs. Configure only what you need, and the app will generate appropriate commands.
</Info>

    - **Enable Secret**: Privilege mode password

### ② DHCP

**Purpose:** Define DHCP pools for VLANs (Layer 3 switches only)

<Warning>
DHCP pools are **only deployed on Layer 3 switches**. Layer 2 switches rely on the router (R1) for DHCP.
</Warning>

<Steps>
  <Step title="Create DHCP Pool">
    For each pool, specify:
    - **Pool Name**: e.g., "VLAN10_POOL"
    - **Network**: e.g., "10.10.10.0"
    - **Subnet Mask**: e.g., "255.255.255.0"
    - **Default Gateway**: e.g., "10.10.10.1" (will be the SVI IP)
  </Step>

  <Step title="Gateway Exclusion">
    The app automatically excludes the gateway IP from the DHCP range:

    ```python
    if is_l3:
        for pool in dhcp_pools:
            cmds.append(f"ip dhcp excluded-address {pool['gw']}")
            cmds += [
                f"ip dhcp pool {pool['name']}",
                f"network {pool['net']} {pool['mask']}",
                f"default-router {pool['gw']}",
                "dns-server 8.8.8.8",
                "exit"
            ]
    ```

    Source: [command_builder.py:115-127](file:///home/daytona/workspace/source/core/command_builder.py#L115-L127)
  </Step>
</Steps>

### ⑤ QoS MQC (Quality of Service)

**Purpose:** Configure Modular QoS CLI (class-map, policy-map, service-policy)

<Steps>
  <Step title="Define Class-Maps">
    Classify traffic by:
    - **DSCP values** (e.g., EF, AF41)
    - **IP addresses** (source/destination)
    - **ACLs** (custom matching)

    ```python
    for cls in qos_classes:
        cmds += [
            f"class-map {cls['match_type']} {cls['name']}",
            f"match {cls['criteria']} {cls['value']}",
            "exit"
        ]
    ```

    Source: [command_builder.py:224-231](file:///home/daytona/workspace/source/core/command_builder.py#L224-L231)
  </Step>

  <Step title="Create Policy-Map">
    Define actions for each traffic class:

    **Available Actions:**
    - `priority` - High-priority queue (voice/video)
    - `bandwidth` - Guaranteed minimum bandwidth
    - `police rate` - Rate limit (drop excess)
    - `shape average` - Traffic shaping (buffer + delay)
    - `set dscp` - Re-mark DSCP value

    ```python
    if pol_entries and pol_name:
        cmds.append(f"policy-map {pol_name}")
        for pe in pol_entries:
            cmds.append(f"class {pe['class']}")
            a, v = pe['action'], pe['value']
            if   a == "police rate":     cmds += [f"police rate {v} bps", "exit"]
            elif a == "shape average":   cmds += [f"shape average {v}",   "exit"]
            elif a == "set dscp":        cmds += [f"set dscp {v}",        "exit"]
            else:                        cmds += [f"{a} {v}",             "exit"]
        cmds.append("exit")
    ```

    Source: [command_builder.py:233-249](file:///home/daytona/workspace/source/core/command_builder.py#L233-L249)
  </Step>

  <Step title="Apply Service-Policy">
    Attach policy-map to interfaces:
    - **Interface**: e.g., "GigabitEthernet0/1"
    - **Direction**: input, output, or both
    - **Policy Name**: The policy-map to apply

    ```python
    for sp in service_policies:
        cmds += [
            f"interface {sp['iface']}",
            f"service-policy {sp['dir']} {sp['policy']}",
            "exit"
        ]
    ```

    Source: [command_builder.py:251-259](file:///home/daytona/workspace/source/core/command_builder.py#L251-L259)

    <Warning>
    QoS is **never applied** to the uplink interface (Gi0/0) to avoid disrupting management traffic.
    </Warning>
  </Step>
</Steps>

<Steps>

    ```python
        """
        Reads running-config and generates 'no ...' commands to clean
        only what this app may have configured previously.

        Protects:
          - SSH, users, VTY, crypto keys (remote access)

          - Static routes and OSPF networks NOT configured by this app
          - VLAN 1 (native, never deleted)
    ```

  <Step title="Execute Configuration">

    The process:
    1. Connect via SSH (Netmiko)
    2. Detect Layer 2 vs Layer 3
    3. Optional: Clean previous config
    4. Generate commands via `build_commands()`

    6. Save config (`write memory`)
    7. Disconnect

    ```python
    def send_cmd_by_cmd(nc, commands: list, log_fn=None):

          and need more time. This method adjusts delay per command.
    ```

### ⑧ Seguridad (Security)

**Purpose:** Configure device access security (v2.2 feature)

<AccordionGroup>
  <Accordion title="Enable Secret (MD5 Type-5)">
    Set a strong enable password using irreversible MD5 hashing:

    ```cisco
    enable secret <password>
    ```

    <Note>
    More secure than `enable password` (Type 7), which uses reversible encryption.
    </Note>
  </Accordion>

  <Accordion title="Login Block-For (Brute-Force Protection)">
    Automatically block SSH access after failed login attempts:

    ```cisco
    login block-for 120 attempts 5 within 60
    login on-failure log
    ```

    **Configuration:**
    - **Attempts:** Max failed logins (e.g., 5)
    - **Within:** Time window in seconds (e.g., 60)
    - **Block For:** Lockout duration in seconds (e.g., 120)

    **Behavior:** If 5 failed attempts occur within 60 seconds, VTY/SSH access is blocked for 120 seconds and logged to syslog.
  </Accordion>

  <Accordion title="Banner MOTD">
    Display a message before login prompt:

    ```cisco
    banner motd #<text>#
    ```

    - Shown on SSH and console login
    - Includes pre-loaded legal warning
    - Delimiter `#` is automatically replaced if found in text
  </Accordion>
</AccordionGroup>

**Security Commands Application:**

```python
# Applied AT THE END to avoid closing active SSH session
sec_cmds = build_security_commands(
    enable_pw   = enable_pw,
    attempts    = login_attempts,
    window      = login_window,

    banner_text = banner_text,
)
if sec_cmds:
    if log_fn:
        log_fn(f"[Seguridad] {len(sec_cmds)} comandos de seguridad...")
```

### ⑨ GRE/IPsec

**Purpose:** Configure encrypted GRE tunnels over IPsec (v2.2 feature)

#### Why GRE + IPsec?

<Info>
IPsec alone cannot transport multicast/broadcast traffic (required for OSPF, EIGRP).

GRE creates a point-to-point tunnel that supports multicast, then IPsec encrypts the entire GRE tunnel.
</Info>

From the README:
> "IPsec no puede transportar tráfico multicast/broadcast por sí solo (necesario para OSPF, EIGRP). GRE crea un túnel punto a punto que sí lo permite, y luego IPsec cifra ese túnel completo."

Source: [README.md:94-99](file:///home/daytona/workspace/source/README.md#L94-L99)

#### Configuration

**GRE Tunnel Parameters:**
- **Tunnel Interface**: e.g., Tunnel0
- **Source**: Local interface or IP
- **Destination**: Remote peer IP
- **Tunnel IP**: IP address for the tunnel interface

**IPsec Parameters:**
- **IKE (Phase 1)**: IKEv1 with DH Group 2 (compatible with vios_l2 15.2)
- **IPsec (Phase 2)**: ESP with AES-256 encryption
- **Pre-Shared Key**: Shared secret for authentication

**Command Generation:**

```python
if gre_tunnels:
    gre_cmds = build_gre_ipsec_commands(gre_tunnels)
    if log_fn:
        log_fn(f"[GRE/IPsec] {len(gre_cmds)} comandos de túnel...")
    cmds += gre_cmds
```

Source: [command_builder.py:277-284](file:///home/daytona/workspace/source/core/command_builder.py#L277-L284)

<Note>
Tunnels are configured **after routes** so the system can properly resolve tunnel source interfaces.
</Note>

## Automatic Protections

The app includes several safety mechanisms:

| Protection | Detail |
|------------|--------|
| `GigabitEthernet0/0` | Never modified (management uplink) |
| SSH / VTY / crypto key | Never touched (preserves remote access) |
| Layer 2 switches | Automatically skip `ip routing` and IP SVIs |
| Cleanup | Only removes what the app configured; preserves existing routes and OSPF |
| Enable secret | Applied **at the end** to avoid closing active SSH session |

## Troubleshooting

<AccordionGroup>
  <Accordion title="Authentication Failures">
    **Symptoms:**
    ```
    ✘ SW-CDMX — Error de Autenticación
    ```

    **Solutions:**
    - Verify username/password in Tab ①
    - Check enable secret is correct
    - Ensure user has privilege level 15
    - Test manually: `ssh user@switch-ip`
  </Accordion>

  <Accordion title="Commands Timing Out">
    **Symptoms:**
    ```
    ✘ SW-CDMX — Timeout
    ```

    **Solutions:**
    - Increase `global_delay_factor` in `connector.py`
    - Check network latency
    - Verify switch isn't overloaded (`show processes cpu`)
    - Reduce number of simultaneous deployments
  </Accordion>

  <Accordion title="Invalid Commands on L2 Switch">
    **Symptoms:**
    ```
    % Invalid input detected at '^' marker.
    ```

    **Solutions:**
    - Verify Layer 2/3 detection in logs
    - Check `show ip route connected` output manually
    - May need to disable routing features in Tab ④
  </Accordion>

  <Accordion title="Configuration Not Saved">
    **Symptoms:**
    - Config works but disappears after reboot

    **Solutions:**
    - Check logs for `write memory` command
    - Manually verify: `show startup-config`
    - May need to manually: `copy running-config startup-config`
  </Accordion>
</AccordionGroup>
