La función de **AWS S3 Backup** permite subir automáticamente el `running-config` de switches seleccionados a un bucket S3, con cifrado en reposo y estructura de rutas organizada.

## Instalación de boto3

La función requiere la librería `boto3` (AWS SDK para Python).

```bash
pip install boto3 --break-system-packages
```

<Info>
  El flag `--break-system-packages` es necesario en sistemas con Python gestionado por el SO (Debian/Ubuntu).
</Info>

## Estructura de rutas en S3

Los backups se organizan automáticamente:

```
<prefijo>/<nombre-sucursal>/<YYYY-MM-DD_HH-MM-SS>.txt
```

### Ejemplos

| Prefijo | Sucursal | Timestamp | Ruta completa |
|---------|----------|-----------|---------------|
| `backups/switches/` | `SW-CDMX` | `2025-04-10_14-30-00` | `backups/switches/SW-CDMX/2025-04-10_14-30-00.txt` |
| `prod/configs/` | `SW-MTY` | `2025-04-15_09-15-30` | `prod/configs/SW-MTY/2025-04-15_09-15-30.txt` |
| `` (vacío) | `SW-GDL` | `2025-04-20_18-45-10` | `SW-GDL/2025-04-20_18-45-10.txt` |

## Cifrado SSE-AES256

Todos los objetos se cifran automáticamente en reposo con **SSE-S3** (Server-Side Encryption con claves gestionadas por AWS).

### ¿Qué es SSE-AES256?

- **SSE**: Server-Side Encryption (cifrado en el servidor)
- **AES256**: Advanced Encryption Standard con clave de 256 bits
- **Gestionado por AWS**: Las claves de cifrado son administradas automáticamente por S3

Código en `ui/tab_exec_backup.py:626-632`:

```python
client.put_object(
    Bucket               = bucket,
    Key                  = s3_key,
    Body                 = content,
    ContentType          = "text/plain; charset=utf-8",
    ServerSideEncryption = "AES256",  # cifrado en reposo (SSE-S3)
)
```

<Info>
  El cifrado SSE-AES256 es **gratuito** y se aplica automáticamente. No requiere configuración adicional.
</Info>

### Alternativas de cifrado

Si necesitas control total de las claves de cifrado, puedes modificar el código para usar:

```python
def verify_s3_credentials(app):
    """
    Verifica las credenciales S3 haciendo un 'head_bucket' (operación de bajo costo).
    """
    try:
        client, bucket = _get_s3_client(app)
        # head_bucket: verifica que el bucket existe y que tenemos acceso.
        # No descarga ni sube datos; costo AWS: $0.
        client.head_bucket(Bucket=bucket)
        messagebox.showinfo(
            "S3 OK ✔",
            f"Conexión exitosa al bucket:\n  s3://{bucket}\n\n"
            "Las credenciales son válidas."
        )
    except Exception as e:
        messagebox.showerror("Error S3", f"No se pudo conectar al bucket:\n{e}")
```

## Creación del cliente S3

El cliente boto3 se crea con credenciales explícitas (no usa el perfil `~/.aws/credentials`):

```python
def _get_s3_client(app):
    """
    Crea y retorna un cliente boto3 de S3 con las credenciales ingresadas en la GUI.
    """
    try:
        import boto3
    except ImportError:
        raise ImportError(
            "boto3 no está instalado.\n"
            "Ejecuta en la terminal:\n"
            "  pip install boto3 --break-system-packages"
        )
    
    access_key = app.s3_access_key.get().strip()
    secret_key = app.s3_secret_key.get().strip()
    region     = app.s3_region.get().strip()
    bucket     = app.s3_bucket.get().strip()
    
    if not all([access_key, secret_key, region, bucket]):
        raise ValueError(
            "Completa todos los campos de AWS S3:\n"
            "Access Key ID, Secret Access Key, Región y Bucket."
        )
    
    # Crear cliente con credenciales explícitas
    client = boto3.client(
        "s3",
        region_name          = region,
        aws_access_key_id    = access_key,
        aws_secret_access_key = secret_key,
    )
    return client, bucket
```

## Flujo completo de backup_to_s3

```python
def backup_to_s3(app):
    """
    Sube el running-config de cada sucursal seleccionada a AWS S3.
    
    Flujo por sucursal:
      1. Conectar por SSH y obtener running-config (fetch_running_config).
      2. Construir la clave S3: <prefijo>/<hostname>/<YYYY-MM-DD_HH-MM-SS>.txt
      3. Subir el contenido (string → bytes UTF-8) con put_object.
      4. Registrar resultado en la vista previa.
    """
    if not _validate_sucursales(app): return
    targets = _get_selected_sucursales(app)
    if not targets: return
    
    # Validar credenciales antes de conectar a los switches
    try:
        client, bucket = _get_s3_client(app)
    except (ImportError, ValueError) as e:
        messagebox.showerror("Error S3", str(e))
        return
    
    prefix    = app.s3_prefix.get().strip().rstrip("/")
    timestamp = datetime.now().strftime("%Y-%m-%d_%H-%M-%S")
    results   = []
    
    for sw in targets:
        sw_name = sw['name']
        try:
            # 1. Obtener running-config vía SSH
            raw = fetch_running_config(sw)
            
            # 2. Construir clave S3
            safe_name = sw_name.replace(" ", "_")
            s3_key    = f"{prefix}/{safe_name}/{timestamp}.txt"
            
            # 3. Encabezado con metadatos
            header = (
                f"# Backup generado por Configurador IOSvL2\n"
                f"# Sucursal : {sw_name}  ({sw['ip']})\n"
                f"# Fecha    : {datetime.now().strftime('%Y-%m-%d %H:%M:%S')}\n"
                f"# Bucket   : s3://{bucket}/{s3_key}\n"
                f"{'#'*60}\n\n"
            )
            content = (header + raw).encode("utf-8")
            
            # 4. Subir a S3
            client.put_object(
                Bucket               = bucket,
                Key                  = s3_key,
                Body                 = content,
                ContentType          = "text/plain; charset=utf-8",
                ServerSideEncryption = "AES256",  # cifrado en reposo (SSE-S3)
            )
            msg = f"  ✔ {sw_name} → s3://{bucket}/{s3_key}"
            results.append(msg)
            
        except Exception as e:
            msg = f"  ✘ {sw_name} → Error: {e}"
            results.append(msg)
    
    # Mostrar resultados
    summary = "\n".join(results)
    _show_in_preview(app, f"=== Backup S3 — {timestamp} ===\n\n{summary}")
    
    ok_count   = sum(1 for r in results if "✔" in r)
    fail_count = len(results) - ok_count
    messagebox.showinfo(
        "Backup S3 completado",
        f"Bucket: s3://{bucket}\n\n"
        f"✔ Exitosos : {ok_count}\n"
        f"✘ Fallidos : {fail_count}\n\n"
        + summary
    )
```

## Errores comunes

### boto3 no instalado

```
boto3 no está instalado.
Ejecuta en la terminal:
  pip install boto3 --break-system-packages
```

**Solución**: Instala boto3 según el mensaje.

### Credenciales inválidas

```
No se pudo conectar al bucket:
An error occurred (InvalidAccessKeyId) when calling the HeadBucket operation: 
The AWS Access Key Id you provided does not exist in our records.
```

**Solución**: Verifica Access Key ID y Secret Access Key en la consola IAM.

### Bucket no encontrado

```
No se pudo conectar al bucket:
An error occurred (NoSuchBucket) when calling the HeadBucket operation: 
The specified bucket does not exist
```

**Solución**: 
- Verifica que el bucket existe en la región especificada
- Crea el bucket desde la consola S3 si no existe

### Permisos insuficientes

```
An error occurred (AccessDenied) when calling the PutObject operation: 
Access Denied
```

**Solución**: Adjunta la política de IAM con permisos `s3:PutObject` al usuario.

## Seguridad

### Buenas prácticas

1. **Rotación de credenciales**: Cambia Access Keys cada 90 días
2. **Política de bucket**: Habilita bloqueo de acceso público
3. **Versionado**: Activa S3 Versioning para recuperar backups anteriores
4. **Lifecycle policies**: Archiva backups antiguos a Glacier después de 90 días
5. **MFA Delete**: Requiere autenticación multifactor para borrar objetos

### Ejemplo de Bucket Policy

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "DenyUnencryptedObjectUploads",
      "Effect": "Deny",
      "Principal": "*",
      "Action": "s3:PutObject",
      "Resource": "arn:aws:s3:::mi-bucket-backups/*",
      "Condition": {
        "StringNotEquals": {
          "s3:x-amz-server-side-encryption": "AES256"
        }
      }
    }
  ]
}
```

Esta política **fuerza** el uso de cifrado SSE-AES256 en todos los objetos.
